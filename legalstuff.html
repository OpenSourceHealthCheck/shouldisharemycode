Should I Share My Code?

Legal Information

This site is designed to raise the profile of a serious issue for scientific research - the reluctance to share your code and software. We do not store any files you choose to upload. We do not perform any inspection of your files. We do not download any files from your repository.

We don't need to do this, because by having written some code it is already good enough to share. However we do recommend that you have a license and readme as part of your code.

If you would like a more comprehensive health check for your code project, including if it has a license, please visit our main site: healthcheck.io

This site came about from a session at the Collaborations Workshop 2014, organised by the Software Sustainability Institute. The source code for the site is available at GitHub. This site created by James Spencer and Neil Chue Hong, from an original idea by Mark Basham, Arfon Smith and James Spencer.
